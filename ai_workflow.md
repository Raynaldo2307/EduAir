# EduAir AI Workflow

## Project Mission
Build a reliable, secure, Jamaican-focused school management system.

## Core Principles
- No shortcuts
- No copy-paste blindly
- Understand every feature
- Security first
- Offline-first mindset

## How I Use AI
AI is my assistant, not my replacement.

AI may:
- Review code
- Suggest improvements
- Find bugs
- Explain concepts

AI may NOT:
- Make architectural decisions alone
- Write features without review
- Override my judgement

## Development Rules
- All major changes are committed to Git
- Test before push
- Document core systems
- Protect student data

- Every AI-generated code change must:
	•	compile,
	•	be reviewed by me,
	•	and match the docs (like attendance.md).”

Attendance System + Data Integrity
